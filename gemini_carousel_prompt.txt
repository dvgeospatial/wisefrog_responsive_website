DEMO CODE (REFERENCE IMPLEMENTATION)
The demo uses:

HTML structure:
- .carousel-container wrapper
- two buttons .carousel-btn.prev-btn and .carousel-btn.next-btn
- a horizontal <ul class="carousel-track" id="track">
- <li class="card"> items inside the track

CSS structure:
- .carousel-track:
  - display: flex;
  - gap: 16px;
  - overflow-x: auto;
  - scroll-behavior: smooth;
  - scroll-snap-type: x mandatory;
  - width: 100%;
  - padding: 10px 0;
  - hides scrollbars;
  - touch-action: pan-y;
- .card:
  - box-sizing: border-box;
  - scroll-snap-align: start;
  - flex-basis on desktop: flex: 0 0 calc((100% - 48px) / 4);
    → exactly 4 whole cards across, with 3 gaps of 16px.
  - flex-basis on mobile (@media max-width:768px):
    flex: 0 0 calc((100% - 16px) / 2);
    → exactly 2 whole cards across, with 1 gap of 16px.
- .carousel-track.no-smooth:
  - scroll-behavior: auto;
  - scroll-snap-type: none;
- .carousel-track.is-dragging:
  - cursor: grabbing;

JavaScript behavior:
- Grabs track, prevBtn, nextBtn, originalCards.
- getMetrics():
  - reads the first .card
  - reads track gap from computed style
  - computes cardWidth = card.offsetWidth + gap
  - computes setWidth = originalCards.length * cardWidth
- Infinite loop setup:
  - Clones originalCards in reverse order and prepends them.
  - Clones originalCards in normal order and appends them.
  - After a short timeout, centers the scroll position on the middle set:
    - adds .no-smooth
    - sets track.scrollLeft = setWidth
    - forces reflow with track.offsetHeight
    - removes .no-smooth
- Pointer dragging:
  - pointerdown:
    - isDown = true
    - adds .no-smooth and .is-dragging
    - stores startX and scrollLeft
    - setPointerCapture
  - pointermove:
    - if isDown, prevents default and updates scrollLeft based on drag distance
  - pointerup/pointercancel:
    - isDown = false
    - removes .no-smooth and .is-dragging
    - releasePointerCapture
    - calls checkLoop()
- Arrow navigation:
  - nextBtn: scrollBy({ left: cardWidth, behavior: 'smooth' })
  - prevBtn: scrollBy({ left: -cardWidth, behavior: 'smooth' })
- Infinite loop correction:
  - checkLoop():
    - if dragging (isDown) → return
    - recomputes { cardWidth, setWidth }
    - buffer = cardWidth * 1.5
    - if track.scrollLeft <= buffer:
        - add .no-smooth
        - track.scrollLeft += setWidth
        - force reflow
        - remove .no-smooth
      else if track.scrollLeft >= track.scrollWidth - track.clientWidth - buffer:
        - add .no-smooth
        - track.scrollLeft -= setWidth
        - force reflow
        - remove .no-smooth
  - scroll listener:
    - debounces with setTimeout(checkLoop, 150)

This demo produces:
- Desktop: exactly 4 whole cards visible.
- Mobile: exactly 2 whole cards visible.
- Infinite looping in both directions for drag and button scroll.
- No hard left/right edge, thanks to clones and jump logic.
